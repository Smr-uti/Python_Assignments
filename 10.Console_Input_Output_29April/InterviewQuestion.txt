Q1. What does the input() function always return, regardless of what the user types?
:-- In Python, the input() function always returns a string (str), no matter what the user types.

Q2. A student writes the following code expecting the output 30, but gets 1020. Why?
a = input("Enter first number: ")
b = input("Enter second number: ")
print(a + b)
:-- The issue here is that Python's input() function always treats whatever the user types as a string (text), even if they type numbers.
So, when the student inputs 10 and 20:
a becomes the string "10"
b becomes the string "20"
Tt performs string concatenation.
"10" + "20" results in "1020".

How to fix it:
To get the mathematical output of 30, you need to explicitly convert (or "cast") those strings into integers using the int() 
function before adding them together:
Ex-
      a = int(input("Enter first number: "))
      b = int(input("Enter second number: "))
      print(a + b)

Q6. What does id() return in Python? What does the id number represent?
:-- In Python, id() is a built-in function that returns a unique integer identifier for a specific object. Every single value in
Python (numbers, strings, lists, functions, etc.) has its own unique id.
The id() function returns the memory address.

Q9. What is the difference between mutable and immutable data types? Give one example of each.
:-- i] Mutable: You can change, add, or remove elements of the object in-place without changing its memory address (id).
ii] Immutable: You cannot change the contents of the object. Any operation that appears to "change" it actually creates a 
brand-new object in memory, leaving the original unchanged.
Ex. Immutable:(str)
    text = "Hello"
    text[0] = "h"  <-- This will raise a TypeError!
# To "change" it, we must create a new string:
    text = "h" + text[1:]

Ex. Mutable:(list)
    my_list = [1, 2, 3]
    print(id(my_list))  
    my_list[0] = 99     
    print(my_list)      
    print(id(my_list))

Q11. What happens when you try to convert a float string like "10.5" directly using int()? Will it work?
:-- No, it will not work. If w try to pass a string containing a decimal point directly into int(), Python will raise a ValueError.
First, convert the string to a float & then convert that float to an int.

float(): Produces a floating-point object (a number with a decimal component, even if the value is a whole number like 10.0).

Q13. Why is it important to convert input() to int() or float() before doing arithmetic operations?
