Q15. Can a Python function return multiple values? How are they returned and how do you unpack them?
:-- Yes! A Python function can return multiple values — using a comma ( , ) between values.
Unpacking means taking each value into a separate variable from the tuple.
Ex.:   def student():
           return "Ganesh", 85, "A"

       name, marks, grade = student()   # unpack
       print(name)     # Ganesh
       print(marks)    # 85
       print(grade)    # A
Output: Ganesh
        85
        A 

Q16. What are positional arguments? What rule must you follow when passing them?
:-- Positional arguments are values passed to a function in a specific order — the order in which parameters are defined.

Rules follow when passing them:
i] Arguments must be passed in the same order as parameters
ii] Number of arguments must equal number of parameters
iii] 1st argument → 1st parameter, 2nd → 2nd, and so on

Q17. What are keyword arguments? What advantage do they have over positional arguments?
:-- Keyword arguments are values passed to a function by specifying the parameter name along with the value.
Advantages of Keyword arguments:
i] Order does not matter
ii] More readable code
iii] Avoid mistakes

Q18. Can you mix positional and keyword arguments in the same function call? What rule applies?

Q19. What are default arguments? Where must default parameters be placed in the function signature?
:-- Default arguments are values that are already set in the function definition — if no value is passed for that parameter,
the default value is used automatically.
Default parameters must always be placed AFTER non-default parameters.
#  Correct — default after non-default
def student(name, age, city="Pune"):
    print(f"{name}, {age}, {city}")

#  Wrong — default before non-default
def student(name, city="Pune", age):   # SyntaxError!
    print(f"{name}, {age}, {city}")

Q20. What error occurs if you place a default parameter BEFORE a non-default parameter?
:-- If a default parameter is placed before a non-default parameter, Python gives a SyntaxError.

Q21. What is *args? What type does Python store the arguments in inside the function?
:-- *args is a special parameter that allows a function to accept any number of positional arguments — you don't need to know
in advance how many values will be passed.
Python stores all *args values in a tuple.

Q22. What is **kwargs? What type does Python store the keyword arguments in inside the function?
:-- **kwargs is a special parameter that allows a function to accept any number of keyword arguments — you don't need to know 
in advance how many key-value pairs will be passed.
Python stores all **kwargs values in a dictionary.

Q23. What is the difference between *args and **kwargs? When would you use each?
:--         *args                              |                   **kwargs
1. Type: Tuple                                 |1. Type: dictionary
2. Values: Without names                       |2. Values: With names (key-value)
3. Call syntax: func(1, 2, 3)                  |3. Call syntax: func(a=1, b=2)

Use *args when you want to pass multiple values but don't know how many.
Use **kwargs when you want to pass named details but don't know how many.

(regular, default, *args, **kwargs)

Q25. When using *args, can you still access individual arguments by index? Demonstrate with an example.
:-- Yes. *args is stored as a tuple, you can access individual arguments by index — just like a normal tuple.
Ex- def show(*args):
        print(args[0])    # first argument
        print(args[1])    # second argument
        print(args[2])    # third argument

    show("Alice", 20, "Pune")
Output- Alice
        20
        Pune

Q26. When using **kwargs, how do you access individual values by key?
:-- **kwargs is stored as a dictionary, we can access individual values by key — just like a normal dictionary.
Ex-  def student(**kwargs):
         print(kwargs['name'])    # access by key
         print(kwargs['age'])     # access by key
         print(kwargs['city'])    # access by key

     student(name="Ganesh", age=20, city="Pune")
Output- Ganesh
        20
        Pune
