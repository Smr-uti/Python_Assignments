Q1. What is the difference between / and // in Python?
:-- / operator- / divides the first number by the second number and always returns a 
floating-point number (a decimal), even if the numbers divide evenly.

// operator- The // operator divides the first number by the second number and rounds the result
down to the nearest whole integer (towards minus infinity). It chops off the decimal part entirely.

Q23. If a = 10 and later a = 3.14, what happens to the variable type?
:-- a = 10: Python creates an integer object (10) in memory and makes the label a point to it. At this moment,
 type(a) is <class 'int'>.

a = 3.14: Python creates a new float object (3.14) in memory and shifts the label a to point to this new object instead. 
At this moment, type(a) becomes <class 'float'>.

Q24. Why does 10 / 5 return 2.0 instead of 2?
:-- Python's rule is that the / operator always returns a floating-point number (float), even if the numbers divide perfectly.

Q25. Why does int(3.9) return 3 and not 4?
:-- The int() function returns 3 because it performs truncation, not rounding.
When you pass a floating-point number to int(), Python simply chops off everything after the decimal point and keeps only the
whole number. It does not care how close the decimal is to the next whole number.
int(3.9) becomes 3
int(3.1) becomes 3
int(-3.9) becomes -3
