Q6. What is the return type of isalpha(), isdigit(), isalnum(), islower(), isupper(), startswith(), and endswith()?
:-- This means they will always return either True or False.
isalpha(), isdigit(), isalnum(), islower(), isupper(): These are validation methods. They evaluate the entire string 
(or its characters) and return True if the condition is met, and False otherwise.

startswith(), endswith(): These are check methods. They compare the beginning or end of a string against a specified 
substring and return True if a match is found, and False otherwise.

Q7. What does the find() method return when a substring is NOT found in the string?
:-- find()- The find() method returns -1 when the specified substring is not found in the string.

Q8. Is find() case-sensitive? What will "Hello".find("hello") return?
:-- Yes, the find() method is case-sensitive.
It will return -1.
Because "Hello" contains an uppercase 'H' and your search query "hello" starts with a lowercase 'h', Python cannot find
an exact match. Since the substring is not found, find() returns -1 as the indicator of failure.

Q9. What is the return value of find() when the substring IS found?
:-- When the substring is found, the find() method returns the lowest index (integer) where the substring begins within
the string.

Q10. What is the difference between len(), min(), and max() when applied to strings?
:-- i] len()-
Returns the total count of characters (including spaces/symbols).
This function counts how many characters exist in the string.

ii] min()- 
Returns the character with the lowest Unicode value.
Characters have numerical codes (A=65, a=97, etc.). min() returns the character that appears earliest in the character
set.

iii] max()-
Returns the character with the highest Unicode value.
This returns the character that appears latest in the character set.

Q11. Can you call upper() or lower() on an already uppercase or lowercase string? What happens?
:-- Yes, you can absolutely call upper() or lower() on a string that is already in that state.
If you call .upper() on a string that is already uppercase, it returns the same string in uppercase.
If you call .lower() on a string that is already lowercase, it returns the same string in lowercase.
