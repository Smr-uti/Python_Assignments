Q1. Write out the LEGB rule in full. What does each letter stand for? In what order does Python search each level?
:-- L (Local): Inside the current function.
E (Enclosing): In any outer functions (nested scopes).
G (Global): At the top level of the script/module.
B (Built-in): In Python’s pre-defined library of functions and keywords.

Search order for each level:
● Local: Python first looks for the name in the current local scope
● Enclosing: If not found, it moves outward to the scope of any enclosing functions
● Global: If still not found, it checks the global scope
● Built-in: Finally, it searches the built-in scope, which contains all of Python's reserved keywords and standard functions.

:-- Enclosing(E):
The Enclosing (or nonlocal) scope is the scope of any outer function that contains a nested function. It comes into play when you 
have a function defined inside another function; the inner function has access to the variables defined in the outer function's 
local scope.

Ex. : def outer_func():
     message = "I am in the enclosing scope!"  # Enclosing scope variable
    
          def inner_func():
          print(message)  # Accesses the variable from the enclosing scope
    
          inner_func()
    outer_func()
Output: I am in the enclosing scope!

Q4. Can an inner function read a variable from the enclosing function without any keyword? Can it MODIFY that variable? What 
keyword is needed to modify an enclosing variable?
:-- Yes, an inner function can read a variable from its enclosing (outer) function without any special keywords.
It cannot modify that variable directly. 
To modify an immutable variable (or to reassign a variable name) from an enclosing scope, you must use the nonlocal keyword.

Q5. What is the nonlocal keyword? How is it different from the global keyword?
:-- The nonlocal keyword is used in nested functions to refer to a variable that is in the nearest enclosing scope. It does not 
allow you to access variables in the global scope; it specifically looks for a variable in a enclosing function.
The global keyword allows you to modify a variable that is defined at the top-level module scope.

Q8. What happens when the same variable name exists at multiple LEGB levels? Which one does Python use?
:--  When a variable name exists at multiple LEGB levels, Python uses the nearest-scope rule: it searches from the inside out 
(L → E → G → B) and uses the first match it finds.
This means a variable in an inner scope shadows (hides) any variable with the same name in an outer scope.

Q10. Can you call an inner function from outside its enclosing function? What error occurs if you try?
:-- No, you cannot call an inner function directly from outside its enclosing function.
An inner function is local to the scope of the outer function. It does not exist in the global scope, so the Python interpreter 
cannot find it.
If you attempt to call the inner function directly (e.g., inner()), Python will raise a NameError.

Q11. What does locals() return inside an inner function? Does it include variables from the enclosing scope?
:-- The locals() function returns a dictionary containing the local symbol table of that specific function.
No. locals() does not include variables from the enclosing, global, or built-in scopes. It only displays the variables that are 
defined or assigned locally within the current function's scope.

Q15. What is the difference between shadowing in a regular function vs shadowing in a nested function?
:-- 1. Shadowing in a Regular Function (Global vs. Local)
This occurs between the Local scope of a function and the Global scope of the module.
●  Behavior: When you assign a value to a variable inside a function that shares a name with a global variable, you are creating 
a new, independent local variable.
●  The Shadow: The global variable still exists, but it becomes "invisible" or shadowed within that function.
●  Accessing the original: You can only modify the global variable by explicitly using the global keyword.

2. Shadowing in a Nested Function (Enclosing vs. Local)
This occurs between the Local scope of the inner function and the Enclosing scope of the outer function.
●  Behavior: When an inner function defines a variable that shares a name with one in its parent (enclosing) function, it shadows the parent's variable.
●  The Shadow: The outer function's variable is still accessible elsewhere, but within the inner function, only the local version is "seen."
●  Accessing the original: You can only modify the outer (enclosing) variable by using the nonlocal keyword.

Q16. What error is raised when an inner function tries to do enclosing_var += 1 without the nonlocal keyword? Why does this happen?
:-- UnboundLocalError: local variable '...' referenced before assignment
When Python compiles a function, it scans the code to determine the scope of each variable. If an assignment occurs anywhere 
within a function (and += is an assignment operation), Python treats that variable as local to that function by default.
