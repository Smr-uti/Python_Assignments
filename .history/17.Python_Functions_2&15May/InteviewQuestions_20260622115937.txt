Q7.  What is the function naming convention in Python? Give one good example and one bad example.
:-- =Python uses snake_case for function names. This is defined in PEP 8, Python's official style guide.
The rules are:

All lowercase letters
Words separated by underscores _
Name should be descriptive — it should say what the function does
Typically starts with a verb (since functions perform actions)
No spaces, hyphens, or special characters

Q15. Can a Python function return multiple values? How are they returned and how do you unpack them?
:-- Yes! A Python function can return multiple values — using a comma ( , ) between values.
Unpacking means taking each value into a separate variable from the tuple.
Ex.:   def student():
           return "Ganesh", 85, "A"

       name, marks, grade = student()   # unpack
       print(name)     # Ganesh
       print(marks)    # 85
       print(grade)    # A
Output: Ganesh
        85
        A 

Q16. What are positional arguments? What rule must you follow when passing them?
:-- Positional arguments are values passed to a function in a specific order — the order in which parameters are defined.

Rules follow when passing them:
i] Arguments must be passed in the same order as parameters
ii] Number of arguments must equal number of parameters
iii] 1st argument → 1st parameter, 2nd → 2nd, and so on
