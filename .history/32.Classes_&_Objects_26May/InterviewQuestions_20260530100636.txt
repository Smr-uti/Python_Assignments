Q1. What is Object-Oriented Programming (OOP)? How is it different from Procedural Programming?
:-- Object-Oriented Programming (OOP) is a programming paradigm based on the concept of "objects," which can contain data (attributes) 
and code (methods). Instead of focusing on logic and functions, OOP focuses on the objects that developers want to manipulate.
● We can create own custom datatype.

Q9.  What is the __init__ method? When is it called?
:-- The __init__ method is a special instance method used as a constructor for a class. Its primary purpose is to initialize the 
state of an object by assigning values to its attributes when the object is created.
● The __init__ method is called automatically the moment a new instance (object) of a class is created. You do not need to call it
manually; the Python interpreter handles this behind the scenes during the instantiation process.

Q12. What is a method? How is it different from a regular function?
:-- A method is essentially a function that is "attached" to a class. While a regular function lives on its own, a method belongs 
to a class and is designed to interact with the data (attributes) of objects created from that class.

Q13. Can you create multiple objects from the same class? Do they share the same attribute values?
:-- Yes, you can create as many objects as you like from the same class. In fact, that is the primary purpose of a class—it acts 
as a reusable template or factory for creating multiple instances.
● No. While all objects created from the same class share the same structure (they have the same list of attributes defined in 
the class), they store their own individual values for those attributes.
